사용 방법
====================================================================
1. LFS-Tools 폴더와 LFS-launcher.bat 파일을 프로젝트 루트 디렉터리에 복사합니다.

2. LFS-launcher.bat을 실행합니다.

3. "[1] Auto-LFS-Tracker"를 실행합니다.

4. "1. 대용량 파일 LFS 자동 설정"을 선택하여 LFS 등록을 완료합니다.

5. LFS 등록이 완료되면 ".gitattributes 파일을 지금 커밋하시겠습니까?"에서 Y를 눌러 커밋을 진행합니다.
   [참고] 신규 확장자 등록만 원할 경우 여기서 종료해도 됩니다.

6. 런처 메인화면으로 돌아와서 "[2] LFS-Migration"을 진행합니다.

7. 마이그레이션이 완료되면 Enter를 눌러 종료합니다.
   [주의] 현재 Enter 키로 정상 종료되지 않는 버그가 있습니다. 
          응답하지 않으면 콘솔창을 직접 닫아도 됩니다.

8. LFS 등록 및 마이그레이션이 완료되면 GitHub Desktop을 실행하여 
   푸시를 진행하면 정상적으로 완료됩니다.

====================================================================

//주의사항//

[Disconnect 에러 발생 시]
최초 LFS 등록 후 등록되지 않은 대용량 파일 확장자를 커밋/푸시하면 
Disconnect 에러가 발생합니다. 이 경우 다시 마이그레이션을 진행해야 
하므로 새로운 대용량 파일 추가 시 주의하세요.

[팀 협업 시 주의사항]
마이그레이션을 진행하면 기존 커밋 히스토리를 삭제하고 재작성하게 되므로 
공동으로 작업하고 있는 팀원은 브랜치와 상관없이 re-clone을 진행해야 합니다. 
해당 과정에서 기존 작업 중이던 내용이 저장되지 않으니 "꼭" 백업을 필수로 
진행해주세요.

[백업 대상]
- Assets 폴더
- Packages 폴더
- ProjectSettings 폴더
- .gitignore 파일

[Windows 11 호환성]
현재 보안정책에 의해 Windows 11 환경에서 실행이 불가능합니다. 가급적 Windows 10 환경에서 실행해주세요.
